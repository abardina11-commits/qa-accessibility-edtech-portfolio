# Примеры эстимации тестовых задач

## Формула оценки
**Время = Анализ + Подготовка данных + Выполнение + Оформление результатов**

## Примеры для проекта

| Задача | Тип задачи | Описание | Оценка (часы) |
|--------|------------|----------|----------------|
| **EAQ-7** | Smoke-тест главной страницы | Проверить загрузку, кликабельность главных ссылок, работу поиска | 1–2 часа |
| **EAQ-8** | Проверка формы авторизации | Позитивные, негативные сценарии, граничные значения, валидация | 2–3 часа |
| **EAQ-9** | Проверка фильтра каталога | Pairwise-тесты (5 браузеров × 5 ОС × 4 категории × 4 роли → 15–20 тестов) | 3–4 часа |
| **EAQ-13** | Навигация с клавиатуры | Проверка всех интерактивных элементов на главной странице и формах | 2–3 часа |
| **EAQ-16** | Accessibility audit страницы (WAVE + Lighthouse) | Прогнать 3 инструмента, сделать скриншоты, оформить 2–3 баг-репорта | 1.5–2 часа |
| **EAQ-23** | Создание API-коллекции в Postman | Настроить переменные окружения, создать 5 запросов (GET /courses, POST /login и т.д.) | 2–3 часа |
| **EAQ-30** | Установка Python + Selenium | Установка, настройка виртуального окружения, проверка работы WebDriver | 1 час |

## Дополнительные рекомендации
- Добавлять буфер 20–30% на непредвиденные проблемы.
- Для accessibility-задач закладывать время на работу со screen reader (NVDA требует привыкания).
- API-тесты требуют меньше времени, если документация хорошая; больше — если её нет.
