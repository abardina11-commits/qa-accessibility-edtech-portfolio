# Bug ID: BUG-004

## Заголовок
Отсутствует видимый индикатор фокуса при навигации с клавиатуры

## Окружение
- Браузер: Chrome 123
- ОС: Windows 10
- Screen reader: NVDA 2024.1 (опционально)
- URL: https://example-edtech.com/login

## Приоритет
High

## Серьёзность
High

## Стандарт WCAG
2.4.7 Focus Visible (Level AA)

## Шаги для воспроизведения
1. Открыть страницу логина
2. Нажать несколько раз клавишу Tab для перемещения между элементами

## Фактический результат
- Фокус не виден или едва заметен (только точка)
- Нет контрастной рамки вокруг активного элемента

## Ожидаемый результат
- Активный элемент имеет рамку контрастностью ≥ 3:1 по сравнению с фоном
- Фокус виден на всех интерактивных элементах: ссылках, кнопках, полях ввода, чекбоксах

## Частота воспроизведения
100%

## Влияние на пользователя
- Пользователи с нарушениями зрения теряют ориентацию на странице
- Невозможно понять, на каком элементе находится фокус

## Технические детали
Вероятно, стиль `:focus { outline: none; }` убран или заменён на невидимый.

## Рекомендации
- Добавить `:focus { outline: 2px solid #1E88E5; outline-offset: 2px; }`
- Или использовать рамку с контрастностью ≥ 3:1

## Тип дефекта
Accessibility / Keyboard Navigation

EAQ-18 (проверка фокуса / клавиатура)
