# Test Case ID: TC-EAQ-001-03

## Название
Проверка валидации пустых полей при авторизации

## Модуль
Авторизация (Login)

## Приоритет
High

## Предусловия
- Пользователь находится на странице авторизации
- Браузер открыт (Chrome)

## Тестовые данные
- Email: (пусто)
- Пароль: (пусто)

## Шаги
1. Не заполнять поле «Email» и поле «Пароль»
2. Нажать кнопку «Войти»

## Ожидаемый результат
- Отображается сообщение: «Пожалуйста, заполните поле Email» и «Пожалуйста, заполните поле Пароль»
- Кнопка «Войти» неактивна (disabled) или нажатие не отправляет форму
- Фокус перемещается на первое пустое поле (email)

(Заполняется при выполнении)

## Статус
(Not Run / Passed / Failed)

## Тип теста
Functional / Validation

## Связанные требования
REQ-AUTH-003

## Accessibility checks
- Сообщения об ошибках связаны с полями (ARIA describedby)
- Поля подсвечены красным

## Комментарий
Проверяет, что система не отправляет запрос с пустыми полями и выводит понятные пользователю ошибки.
