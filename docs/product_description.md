# Product Description

## 1. Общая информация

**Название продукта:** EdTech Demo Platform  
**Тип:** Веб-приложение  
**Домен:** Образование (EdTech)  
**Цель продукта:** предоставление онлайн-доступа к образовательным курсам и материалам

---

## 2. Целевая аудитория

Основная аудитория:
- взрослые пользователи 35–60+  
- родители, принимающие решения об обучении  
- пользователи с разным уровнем цифровой грамотности  

Особенности аудитории:
- повышенная чувствительность к сложности интерфейса  
- важна понятная навигация и предсказуемость действий  
- возможны когнитивные особенности (внимание, память, скорость восприятия)

---

## 3. Основной функционал

- Регистрация пользователя  
- Авторизация (логин)  
- Просмотр каталога курсов  
- Просмотр карточки курса  
- Запись на курс  
- Работа с личным кабинетом  

---

## 4. Пользовательские сценарии

Ключевые сценарии:

1. Новый пользователь:
   - заходит на сайт  
   - регистрируется  
   - выбирает курс  
   - записывается  

2. Зарегистрированный пользователь:
   - авторизуется  
   - просматривает каталог  
   - продолжает обучение  

---

## 5. Требования к качеству

### Функциональные
- корректная работа всех форм  
- валидная обработка данных  
- стабильная работа основных сценариев  

### Нефункциональные
- удобство использования (usability)  
- доступность (accessibility, WCAG 2.1 AA)  
- кроссбраузерность  
- понятные сообщения об ошибках  

---

## 6. Фокус тестирования

В рамках портфолио основной фокус:

- Accessibility testing (WCAG 2.1)  
- Usability testing  
- Ручное тестирование ключевых сценариев  

Особое внимание:
- читаемость интерфейса  
- логика навигации  
- понятность действий для пользователя  

---

## 7. Инструменты тестирования

- Chrome DevTools  
- Lighthouse  
- WAVE  
- axe DevTools  
- NVDA (screen reader)  

---

## 8. Тестовая среда

- Браузеры: Chrome  
- Тип устройства: Desktop  
- Тестирование на демо/стейджинг среде  

---

## 9. Риски

- сложная навигация для пользователей 50+  
- перегруз интерфейса информацией  
- ошибки в формах (регистрация/логин)  
- несоответствие требованиям доступности  

---

## 10. Ограничения

- тестирование проводится без доступа к исходному коду  
- автоматизация находится в стадии изучения  
- API тестирование — базовый уровень  

---

## 11. Результат

В рамках проекта:
- сформированы тест-кейсы  
- оформлены баг-репорты  
- проведён аудит доступности  
- выявлены проблемы usability
