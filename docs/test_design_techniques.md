# Техники тест-дизайна в проекте

## 1. Классы эквивалентности
- Пример для поля Email: валидный email, невалидный (без @), пустое поле

## 2. Граничные значения
- Длина пароля: 8-20 символов → тесты на 7,8,20,21 символ

## 3. Попарное тестирование (Pairwise)
- Комбинации: браузер × ОС × роль пользователя

## 4. Таблица принятия решений
- Условия: [да/нет] для формы регистрации → результаты

## 5. Предугадывание ошибок
- Чек-лист типичных проблем для форм и навигации
