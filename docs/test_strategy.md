# Test Strategy

## 1. Цель тестирования

Обеспечить качество веб-приложения EdTech Demo Platform через:
- проверку корректности функционала  
- оценку удобства использования (usability)  
- проверку доступности (accessibility, WCAG 2.1 AA)  

Особый фокус — пользователи 35–60+ с разным уровнем цифровой грамотности.

---

## 2. Объект тестирования

Веб-приложение EdTech Demo Platform.

Ключевые модули:
- Регистрация  
- Авторизация  
- Каталог курсов  
- Карточка курса  
- Личный кабинет  

---

## 3. Объем тестирования

### Включено:
- Функциональное тестирование основных пользовательских сценариев  
- Usability тестирование интерфейса  
- Accessibility тестирование (WCAG 2.1 AA)  
- Проверка форм и валидации данных  

### Исключено:
- Нагрузочное тестирование  
- Безопасность (security testing)  
- Глубокое API тестирование  

---

## 4. Типы тестирования

### Функциональное тестирование
- проверка бизнес-логики  
- проверка пользовательских сценариев  
- обработка ошибок  

---

### Usability testing
Фокус:
- понятность интерфейса  
- логичность навигации  
- предсказуемость действий  

Особое внимание:
- минимизация когнитивной нагрузки  
- простота взаимодействия  

---

### Accessibility testing
Проверка:
- контрастности  
- структуры заголовков  
- доступности через клавиатуру  
- работы со screen reader (NVDA)  

Стандарт:
- WCAG 2.1 AA  

---

### Кроссбраузерное тестирование
- Chrome (основной)  

---

## 5. Подход к тестированию

Используется комбинированный подход:

- чек-листы для быстрой проверки  
- тест-кейсы для критичных сценариев  
- исследовательское тестирование (exploratory)  
- тест-дизайн техники  

---

## 6. Тест-дизайн

Применяемые техники:

- Boundary Value Analysis (граничные значения)  
- Pairwise testing (попарное тестирование)  
- Decision Tables (таблицы решений)  

---

## 7. Приоритизация

Приоритет тестирования:

1. Критичные сценарии:
   - регистрация  
   - авторизация  
   - запись на курс  

2. Основной пользовательский путь  

3. Дополнительный функционал  

---

## 8. Тестовая среда

- Браузер: Chrome  
- Устройство: Desktop  
- Среда: demo / staging  

---

## 9. Инструменты

- Chrome DevTools  
- Lighthouse  
- WAVE  
- axe DevTools  
- NVDA  
- Jira  
- GitHub  

---

## 10. Критерии начала тестирования

- доступ к системе  
- определены основные сценарии  
- подготовлены тест-кейсы или чек-листы  

---

## 11. Критерии завершения тестирования

- протестированы основные сценарии  
- задокументированы найденные дефекты  
- проведена проверка доступности  
- оформлены результаты  

---

## 12. Управление дефектами

- каждый дефект оформляется как отдельный bug report  
- используется стандартный шаблон  
- приоритизация: High / Medium / Low  

---

## 13. Риски

- сложный интерфейс для пользователей 50+  
- высокая когнитивная нагрузка  
- ошибки в формах  
- проблемы доступности  

---

## 14. Метрики

- количество тест-кейсов  
- количество найденных дефектов  
- покрытие основных сценариев  
- количество accessibility-проблем  

---

## 15. Результаты

В рамках тестирования:
- подготовлены тест-кейсы  
- оформлены баг-репорты  
- выявлены проблемы usability и accessibility
