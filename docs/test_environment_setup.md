# Настройка тестовой среды

## 1. Объект тестирования
**Демо-сайт:** [https://demoqa.com](https://demoqa.com) (учебный сайт с формами, кнопками, таблицами)  
**Почему выбран:**  
- Не требует логина/пароля  
- Быстро загружается на слабом ноутбуке  
- Содержит все типичные элементы: формы, кнопки, всплывающие окна

> В будущем можно тестировать реальные EdTech-платформы (например, открытые уроки на Stepik, Coursera).

## 2. Аппаратное и программное обеспечение
| Компонент | Характеристика |
|-----------|----------------|
| **Ноутбук** | 6 ГБ ОЗУ, Windows 10 |
| **Браузер** | Google Chrome (последняя версия) |
| **Инструменты** | WAVE, axe DevTools, Lighthouse (встроен), Postman |
| **Дополнительно** | NVDA (скринридер) – по желанию |

## 3. Расширения и настройки браузера
- ✅ **WAVE Evaluation Tool** — установлен  
- ✅ **axe DevTools** — установить из Chrome Web Store  
- ✅ **Lighthouse** — уже в Chrome DevTools (F12 → Lighthouse)  
- ⚠️ Отключить тяжёлые расширения (ускоряет работу)

## 4. Тестовые данные
Все данные хранятся в `examples/test-data/`:
- `valid_users.json` — корректные email/пароли  
- `invalid_users.csv` — данные с ошибками (граничные значения)  
- `test_registration_form.csv` — комбинации для pairwise

## 5. Ограничения (важно для работодателя)
- **Слабый интернет** → использовать легковесные инструменты, не грузить видео  
- **Мало времени (≤5 часов/день)** → тестировать по одному сценарию за раз  
- **Нет доступа к бэкенду** → только frontend и API, если есть открытые эндпоинты

## 6. Процесс запуска тестирования
1. Открыть браузер → перейти на демо-сайт  
2. Включить расширение WAVE (иконка в правом верхнем углу)  
3. Открыть DevTools (F12) → вкладка Lighthouse  
4. Выполнить тест-кейс  
5. При баге → скриншот (Win+Shift+S) → сохранить в `assets/screenshots/`  
6. Оформить баг-репорт по шаблону `tools/bug_report_template.md`

## 7. Коммуникационные предпочтения
- **Только асинхронное общение** (чат, email, комментарии в Jira/GitHub) — письменный текст воспринимаю лучше, в устной речи иногда теряюсь.
- **Гибкий график** — работаю не в московском часовом поясе, ответы с задержкой до 12 часов.
