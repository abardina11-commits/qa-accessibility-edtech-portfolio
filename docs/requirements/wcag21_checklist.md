# Чек-лист соответствия WCAG 2.1 (уровни A и AA)

## 1. Воспринимаемость (Perceivable)

### 1.1 Альтернативный текст (Level A)
- [ ] Все информативные изображения имеют `alt`-текст
- [ ] Декоративные изображения имеют пустой `alt=""`
- [ ] Сложная графика (`canvas`, SVG) сопровождается текстовым описанием

### 1.3 Адаптивность контента (Level A)
- [ ] Правильная иерархия заголовков (`h1` → `h2` → `h3`, без пропусков)
- [ ] Формы имеют связанные `<label>` (не `placeholder` как замена)
- [ ] Таблицы с данными имеют `<th>` и атрибут `scope`
- [ ] Списки оформлены семантическими тегами `<ul>`, `<ol>`

### 1.4 Различимость

#### 1.4.3 Контрастность (Level AA)
- [ ] Обычный текст: контрастность ≥ 4.5:1
- [ ] Крупный текст (18pt+): контрастность ≥ 3:1
- [ ] UI-элементы (кнопки, границы полей ввода) имеют контрастность ≥ 3:1
- [ ] Состояния `hover`, `focus`, `active` сохраняют контрастность

#### 1.4.4 Масштабирование текста (Level AA)
- [ ] При масштабировании 200% контент не обрезается и не перекрывается
- [ ] Нет горизонтальной прокрутки при масштабе 200%

#### 1.4.10 Адаптивная верстка (Level AA)
- [ ] Медиа-запросы для mobile (320px ↔ 1280px)
- [ ] Кнопки и ссылки имеют размер кликабельной области (touch targets ≥ 44×44px)

#### 1.4.11 Контрастность элементов (Level AA)
- [ ] Границы полей ввода имеют контрастность ≥ 3:1 по сравнению с фоном
- [ ] Визуальные индикаторы состояний (выделение строки таблицы) различимы

#### 1.4.12 Интерлиньяж (Level AA) – опционально
- [ ] Настройка межстрочного интервала до 1.5
- [ ] Настройка интервала между абзацами

## 2. Управляемость (Operable)

### 2.1 Доступность с клавиатуры (Level A)
- [ ] Все интерактивные элементы доступны через `Tab`
- [ ] Нет «ловушек фокуса»: всегда можно уйти с элемента клавишей `Tab`
- [ ] Выпадающие меню, модальные окна, кастомные select управляются со стрелками

### 2.1.2 Без ловушек фокуса (Level A) – уточнение
- [ ] Фокус цикличен, но не застревает
- [ ] Клавиша `Esc` закрывает модальные окна и возвращает фокус на триггер

## 3. Понятность (Understandable)

### 3.3.1 Идентификация ошибок (Level A)
- [ ] При ошибке валидации сообщение понятное, подсвечивается проблемное поле
- [ ] Сообщение об ошибке находится рядом с проблемным полем

### 3.3.3 Предотвращение ошибок (Level AA)
- [ ] Критичные действия (удаление аккаунта, оплата) требуют подтверждения
- [ ] Для обязательных полей указано "(обязательно)"
